프로젝트 코드를 실제로 정리하고 수정해줘. 리포트만 쓰지 말고, 아래 4단계를 순서대로 **직접 실행**해.

---

## 실행 순서

### Step 1 — Unused Code 제거
1. `src/` 전체를 스캔해서 사용되지 않는 `import`, 변수, 함수를 찾아.
2. 해당 항목을 **실제로 삭제**해. (주석 처리 금지, 완전 제거)
3. 삭제 후 빌드/타입 오류가 생기지 않는지 확인해.

### Step 2 — Architecture 재정비
1. `src/` 내 파일 목록을 전체 확인해.
2. 현재 폴더 구조와 CLAUDE.md 아키텍처 원칙을 비교해.
3. 역할이 섞인 파일이 있으면 `services/`, `utils/`, `types/` 등 적절한 폴더로 이동해.
4. 파일 이동 시 해당 파일을 import하는 모든 곳의 경로를 자동으로 업데이트해. (경로 깨짐 금지)

### Step 3 — 중복 로직 추출 (Refactor)
1. 여러 파일에 걸쳐 반복되는 로직 패턴을 찾아. (예: 좌표 보정, DOM 유틸, 텍스트 정규화 등)
2. 반복 로직을 `src/utils/` 아래 공통 함수로 추출해.
3. 기존 호출 위치를 새 공통 함수로 교체하고 기존 중복 코드를 삭제해.

### Step 4 — 네이밍 일관성 (Consistency)
1. 변수명·함수명이 camelCase 규칙에 어긋난 곳을 찾아.
2. 이름을 일관성 있게 수정하고, 해당 이름을 참조하는 모든 위치도 함께 업데이트해.

---

## 완료 후 출력 형식

각 단계별로 다음을 보고해:
- **변경된 파일 목록** (경로 포함)
- **변경 내용 요약** (무엇을 왜 바꿨는지 한 줄씩)
- **주의사항** (타입 오류 가능성, 수동 확인 필요 항목 등)

최종적으로 `/cm` 커밋 메시지 추천을 실행해줘.
